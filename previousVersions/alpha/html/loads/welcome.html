Liefs Layout Manager is Javascript Code that manages the locations of elements on the page.

(JQuery NOT Required -> but plays friendly)

It controls the (.left .top .width .length .position .visible) style properties of element. By doing so, it is able to accomplish these goals:
  1. Define a Layout for a page.
    1. Allow Any Number of windows (divs, img, etc...) on a page
    2. Allow the dimension of these windows to be defined as either
      1. A fixed number of Pixels (ie "120px")
      2. A Percentage of the screen (ie "50%")
  2. Create a set of rules: When the screen is this size, use this layout.
For example THIS VERY PAGE (Ctrl-u to view source)
Started from this basic template: (But Uses all the Frills) 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <script src="/js/basic1.js"></script> <!-- this is the REQUIRED CODE for liefs to work! -->
<!--<script src="http://liefs.paperplane.io/js/basic1.js"></script> or use this -->
<script>
V("Container", 4,    // This Page Container is Vertical (Header on top, [menu, Main] on bottom) 4px margins
        I("header", "40px"),   // This page has a "Header" Item that is 40px High
        h("menuMain", "100%",  // The Bottom Item is a Horizontal Container (100% of rest) of Vertical Container
                I("menu", "200px"),  // This new Bottom Part Consists of a menu Item on the left (200px wide)
                I("welcome", "100%") // And a Content Item to fill what Tree does not. (100% of rest)
        )
)
use("Container"); // set Container for Handler to use
startHandler();   // Now, watches for screen resize, if so, change style properties of divs
</script>
<body>
    <div id="header">{Docs Liefs_Layout-Manager=}</div>        <!-- Used TrueTypeFont Here standard stuff           -->
    <div id="menu" urlload="/html/blocks/menu.html"></div>     <!-- liefs layout includes urlload in the tags       -->
    <div id="welcome" urlload="/html/docs/welcome.html"></div> <!-- This works recursivly until nothing left to load-->
</body>
</html>   <!-- urlload is a tool for development - I recommend Manually resolving these at point of production -->
